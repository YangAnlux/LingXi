# 管理系统局域网访问配置修改指南

## 1. 修改目标

解决管理系统只能通过 `localhost` 访问，局域网内其他电脑无法连接的问题。

修改完成后，局域网内其他电脑应通过部署电脑的局域网 IP 访问：

```text
管理后台：http://部署电脑IP
商城前端：http://部署电脑IP:81
后端接口：http://部署电脑IP:8080
```

例如部署电脑 IP 是 `192.168.1.100`：

```text
http://192.168.1.100
http://192.168.1.100:81
http://192.168.1.100:8080/actuator/health
```

> 不要把 `localhost` 替换成组长电脑的固定 IP 并提交到公共代码中。推荐使用“前端同源请求 + Nginx反向代理”，这样每台电脑不需要修改成不同IP。

### 2.3 查询部署电脑局域网 IP

```powershell
Get-NetIPAddress -AddressFamily IPv4 |
Where-Object {
    $_.IPAddress -notlike "127.*" -and
    $_.IPAddress -notlike "169.254.*" -and
    $_.InterfaceAlias -notmatch "vEthernet|Loopback|VMware"
} |
Select-Object InterfaceAlias, IPAddress
```

优先使用当前正在联网的 `WLAN` 或“以太网”对应地址。

也可以使用：

```powershell
ipconfig
```

找到当前网卡的“IPv4 地址”。

## 4. 修改管理后台 Nginx 配置

打开：

```text
product/product/docker-compose/nginx/web.conf
```

将：

```nginx
listen 80;
server_name localhost;
```

修改为：

```nginx
listen 0.0.0.0:80 default_server;
server_name _;
```

确认管理端接口代理保持如下形式：

```nginx
location /admin-api/ {
    proxy_pass http://server:8080/admin-api/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

这里的 `server` 是 Docker Compose 中后端服务名，不要改成 `localhost`。

## 5. 修改商城 Nginx 配置

打开：

```text
product/product/docker-compose/nginx/mall.conf
```

将：

```nginx
listen 80;
server_name localhost;
```

修改为：

```nginx
listen 0.0.0.0:80 default_server;
server_name _;
```

商城接口代理应保留：

```nginx
location /app-api/ {
    proxy_pass http://server:8080/app-api/;
}
```

## 6. 修改 Docker Compose 端口绑定

打开：

```text
product/product/docker-compose/docker-compose.yml
```

将后端端口改为：

```yaml
server:
  ports:
    - "0.0.0.0:${SERVER_PORT:-8080}:8080"
```

将管理后台端口改为：

```yaml
web:
  ports:
    - "0.0.0.0:${WEB_PORT:-80}:80"
```

将商城端口改为：

```yaml
mall:
  ports:
    - "0.0.0.0:${MALL_PORT:-81}:80"
```

说明：

- `0.0.0.0` 表示监听电脑所有网卡；
- `80` 是管理后台端口；
- `81` 是商城前端端口；
- `8080` 是后端接口端口；
- 不要把数据库、Redis和RabbitMQ端口开放给不可信网络。

## 9. 配置 Windows 防火墙

使用“管理员身份”打开 PowerShell，执行：

```powershell
New-NetFirewallRule `
  -DisplayName "MITEDTSM-Web-LAN" `
  -Direction Inbound `
  -Action Allow `
  -Protocol TCP `
  -LocalPort 80 `
  -RemoteAddress LocalSubnet `
  -Profile Any

New-NetFirewallRule `
  -DisplayName "MITEDTSM-Mall-LAN" `
  -Direction Inbound `
  -Action Allow `
  -Protocol TCP `
  -LocalPort 81 `
  -RemoteAddress LocalSubnet `
  -Profile Any

New-NetFirewallRule `
  -DisplayName "MITEDTSM-Server-LAN" `
  -Direction Inbound `
  -Action Allow `
  -Protocol TCP `
  -LocalPort 8080 `
  -RemoteAddress LocalSubnet `
  -Profile Any
```

`LocalSubnet` 表示只允许本地局域网访问，不直接向整个互联网开放。

查看规则：

```powershell
Get-NetFirewallRule -DisplayName "MITEDTSM-*-LAN" |
Get-NetFirewallPortFilter |
Select-Object Protocol, LocalPort
```

## 10. 本机验证

假设部署电脑 IP 为 `192.168.1.100`：

```powershell
Invoke-WebRequest "http://192.168.1.100" -UseBasicParsing
Invoke-RestMethod "http://192.168.1.100:8080/actuator/health"
```

预期结果：

- 前端状态码为 `200`；
- 后端返回 `status: UP`。

验证前端通过 Nginx 访问后端：

```powershell
Invoke-WebRequest `
  "http://192.168.1.100/admin-api/system/tenant/get-by-website?website=http%3A%2F%2F192.168.1.100" `
  -UseBasicParsing
```

接口应返回 HTTP 200。

## 11. 其他电脑验证

确保两台电脑：

1. 连接同一个路由器或校园局域网；
2. 可以互相访问；
3. 部署电脑没有关闭 Docker Desktop；
4. Windows 防火墙规则已经添加。

在其他电脑浏览器访问：

```text
http://部署电脑IP
```

如果出现旧接口地址或白屏，按：

```text
Ctrl + F5
```

或者清除该网站缓存后重新打开。

## 12. 常见问题

### 12.1 前端可以打开，但登录接口失败

打开浏览器开发者工具，检查 Network 请求。

正确请求应类似：

```text
http://部署电脑IP/admin-api/system/auth/login
```

如果仍然请求：

```text
http://localhost:8080/admin-api/...
```

说明没有重新执行 `pnpm run build:prod`，或者浏览器仍在使用旧缓存。

### 12.2 其他电脑完全打不开

依次检查：

```powershell
docker compose ps
netstat -ano | findstr ":80 "
netstat -ano | findstr ":8080 "
```

并确认防火墙入站规则已添加。

### 12.3 IP 经常变化

可以在路由器中为部署电脑设置 DHCP 地址保留，或者给电脑配置固定局域网 IP。

不要每次把新 IP 写入前端代码；同源代理方案不依赖固定 IP。

### 12.4 Docker 显示端口被占用

检查占用程序：

```powershell
Get-NetTCPConnection -LocalPort 80,81,8080 -ErrorAction SilentlyContinue |
Select-Object LocalAddress, LocalPort, State, OwningProcess
```

如果端口确实被其他程序占用，可以在 `docker-compose/.env` 中调整：

```dotenv
WEB_PORT=8088
MALL_PORT=8089
SERVER_PORT=8090
```

修改后访问地址也要使用新端口。

建议提交：

```text

docker-compose/nginx/web.conf
docker-compose/nginx/mall.conf
```

```text
target
```
