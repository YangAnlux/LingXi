## 2. 文档基础

### 2.1 判定优先级原则

| 优先级 | 依据 | 说明 |
| :--- | :--- | :--- |
| **P0** | 源码 | 实际代码实现为最高优先级，反映系统真实状态 |
| **P1** | 验收要求 | 明确的功能验收标准，定义系统应达到的目标 |
| **P2** | 原型 | 设计原型，反映预期的业务流程和用户体验 |
| **P3** | 文档 | 项目文档，作为参考辅助说明 |

### 2.2 完成度/状态定义

| 状态 | 定义 | 判断标准 |
| :--- | :--- | :--- |
| **已具备** | 功能完整实现，可正常使用 | 有完整的 Controller/Service/Mapper/DO，前端有 API 和页面，数据库有表 |
| **部分具备** | 功能基本实现，但存在缺陷或不完整 | 核心功能可用，但缺少部分辅助功能或存在已知问题 |
| **可复用** | 其他模块已实现相同或相似功能，可直接复用 | 跨模块存在相同能力，无需重复开发 |
| **缺失** | 功能完全未实现 | 无相关代码、页面和数据库表 |
| **待确认** | 需求不明确或需要进一步验证 | 原型或文档描述不清晰，需与业务方确认 |

### 2.3 判断标准

| 维度 | 标准 | 证据来源 |
| :--- | :--- | :--- |
| **代码量** | 统计 Java 文件数、Vue 文件数、API 接口数 | 文件系统统计 |
| **接口数** | 统计 Controller 中的 API 方法数量 | 源码分析 |
| **页面数** | 统计 Vue 组件数量 | 文件系统统计 |
| **表数量** | 统计数据库表数量 | SQL 查询 |
| **模块完整性** | 检查是否包含 CRUD 完整操作 | 源码分析 |
| **链路完整性** | 检查业务主链各节点是否连通 | 业务流程分析 |

## 3. 原型分析

### 3.1 原型模块目标

#### 3.1.1 销售自动化模块

**线索管理**
- 目标：实现线索的全生命周期管理
- 核心要求：
  - 支持线索录入、分配、跟进、转化
  - 支持线索来源、状态、优先级管理
  - 支持线索转化为客户

**客户管理**
- 目标：实现客户信息的完整管理
- 核心要求：
  - 支持客户基本信息管理
  - 支持客户分类、标签管理
  - 支持客户池管理和分配规则

**联系人管理**
- 目标：实现联系人与客户/商机的关联管理
- 核心要求：
  - 支持联系人信息管理
  - 支持联系人与客户、商机的关联
  - 支持联系人跟进记录

**商机管理**
- 目标：实现商机的全流程管理
- 核心要求：
  - 支持商机创建、阶段管理、预估金额
  - 支持商机状态变更和跟进记录
  - 支持商机关联产品和联系人

**合同管理**
- 目标：实现合同签订和审批管理
- 核心要求：
  - 支持合同创建、审批、归档
  - 支持合同关联商机和产品
  - 支持合同金额、期限管理

**回款管理**
- 目标：实现回款跟踪和计划管理
- 核心要求：
  - 支持回款记录创建和跟踪
  - 支持回款计划制定和提醒
  - 支持回款与合同关联

#### 3.1.2 营销工具模块

**群发管理**
- 目标：实现批量短信/邮件发送能力
- 核心要求：
  - 支持短信和邮件两种群发类型
  - 支持从客户列表中选择目标客户
  - 支持模板消息发送
  - 记录发送状态和统计结果

#### 3.1.3 数据分析模块

**数据统计**
- 目标：实现销售数据的可视化分析
- 核心要求：
  - 支持销售漏斗分析
  - 支持业绩统计和趋势分析
  - 支持客户分析和报表导出

#### 3.1.4 服务支持模块

**工单管理（缺失）**
- 目标：实现客户服务工单管理
- 核心要求：
  - 支持工单创建、分配、处理、关闭
  - 支持工单优先级和SLA管理
  - 支持工单与客户关联

### 3.2 核心业务主链

```
线索（clue）→ 客户（customer）→ 联系人（contact）→ 商机（business）→ 合同（contract）→ 审批（bpm）→ 回款（receivable）→ 工单（缺失）→ 统计（statistics）
                    ↓                              ↓                                                                      ↓
               跟进记录（followup）           产品管理（product）                                                      回款计划（receivable_plan）
                    ↓
               群发管理（broadcast）
```

### 3.3 原型页面和交互覆盖

#### 3.3.1 线索管理页面

**页面结构**
- 搜索区域：线索编号、名称、来源、状态、负责人筛选
- 列表区域：编号、名称、来源、状态、负责人、操作
- 操作区域：新增、编辑、删除、转化为客户

**交互流程**
```
点击"新增" → 弹出表单 → 填写线索信息 → 保存
点击"转化" → 确认转化 → 创建客户记录 → 线索状态变更
点击"删除" → 确认删除 → 删除记录
```

#### 3.3.2 客户管理页面

**页面结构**
- 搜索区域：客户名称、行业、等级、状态筛选
- 列表区域：名称、行业、等级、联系人、状态、操作
- 操作区域：新增、编辑、删除、查看详情

**交互流程**
```
点击"新增" → 弹出表单 → 填写客户信息 → 保存
点击"详情" → 打开详情页 → 查看客户信息和关联记录
点击"删除" → 确认删除 → 删除记录
```

#### 3.3.3 商机管理页面

**页面结构**

### 4.2 关键业务差距

#### 4.2.1 审批未闭环

**问题**：合同审批流程与业务流程未完全打通

**现状**：
- BPM 模块已集成
- 合同审批接口已存在
- 审批状态未与合同状态联动

**影响**：
- 合同状态无法自动更新
- 审批结果无法追溯
- 业务流程中断

**建议**：
- 建立审批回调机制
- 实现审批状态与合同状态联动
- 添加审批历史记录

#### 4.2.2 回款提醒缺失

**问题**：回款到期前无提醒机制

**现状**：
- 回款计划已实现
- 无到期提醒功能

**影响**：
- 回款逾期风险增加
- 财务人员无法及时催收
- 现金流管理困难

**建议**：
- 实现回款到期提醒（提前3天、7天）
- 集成系统通知功能（站内信、短信）
- 添加回款逾期预警

#### 4.2.3 工单管理缺失

**问题**：客户服务工单管理模块未实现

**现状**：
- 无工单相关代码
- 无工单数据库表
- 客户服务需求无法满足

**影响**：
- 客户服务请求无法跟踪
- 服务质量无法监控
- 业务主链断裂（回款→工单→统计）

**建议**：
- 创建工单数据库表
- 实现工单 CRUD 功能
- 实现工单分配和处理流程
- 集成通知功能

#### 4.2.4 自定义报表缺失

**问题**：统计模块缺少自定义报表能力

**现状**：
- 固定统计图表已实现
- 无自定义报表功能

**影响**：
- 无法满足个性化分析需求
- 数据可视化能力有限
- 决策支持能力不足

**建议**：
- 实现自定义报表配置
- 支持多维度数据钻取
- 支持报表导出（Excel/PDF）
