1. 发票号码唯一性校验：创建/编辑时检查 invoice_no 不重复

## 六、存在的缺陷与修复

### 缺陷 1：CrmInvoiceMapper.notInIfPresent() 方法不存在
- **发现时间**：2026-07-14 11:25
- **现象**：Maven 编译失败，`symbol not found: method notInIfPresent`
- **原因**：框架的 `LambdaQueryWrapperX` 不提供 `notInIfPresent()` 方法（与 MyBatis Plus 标准 Wrapper 不同）
- **修复**：改为先查询全量再在内存中 filter 排除，避免依赖不存在的 API
- **影响文件**：[CrmInvoiceMapper.java](../../product/Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/dal/mysql/invoice/CrmInvoiceMapper.java)

### 缺陷 2：Set\<Long\> 无法转换为 Collection\<Integer\>
- **发现时间**：2026-07-14 11:25
- **现象**：编译类型不匹配错误
- **原因**：Service 传入 `Set<Long>` 但 Mapper 参数声明为 `Collection<Integer>`
- **修复**：统一参数类型为 `Collection<Long>`
- **影响文件**：[CrmInvoiceMapper.java](../../product/Server/mitedtsm-module-crm/src/main/java/com/meession/etm/module/crm/dal/mysql/invoice/CrmInvoiceMapper.java)

### 缺陷 3：单元测试 ApplicationContext 加载失败
- **发现时间**：2026-07-14 11:26
- **现象**：SpringBootTest 报错 "No schema scripts found at classpath:/sql/create_tables.sql"
- **原因**：CRM 模块缺少测试用的 DDL 脚本
- **修复**：创建 `src/test/resources/sql/create_tables.sql`（crm_invoice 表）和 `clean.sql`
- **影响文件**：[create_tables.sql](../../product/Server/mitedtsm-module-crm/src/test/resources/sql/create_tables.sql)、[clean.sql](../../product/Server/mitedtsm-module-crm/src/test/resources/sql/clean.sql)

### 缺陷 4：Mockito 参数匹配器误用
- **发现时间**：2026-07-14 11:28
- **现象**：`Misplaced or misused argument matcher` — 尝试 mock `@Resource` 注入的真实 Bean 的方法
- **原因**：`CrmInvoiceMapper` 是真实 Bean（`@Resource`），不能 mock，只能 mock `@MockitoBean` 的外部依赖
- **修复**：重写测试策略，Mapper 使用真实数据插入，仅 mock `CrmNoRedisDAO`、`CrmContractService` 等外部依赖
- **影响文件**：[CrmInvoiceServiceTest.java](../../product/Server/mitedtsm-module-crm/src/test/java/com/meession/etm/module/crm/service/invoice/CrmInvoiceServiceTest.java)

### 缺陷 5：assertServiceException 消息参数不匹配
- **发现时间**：2026-07-14 11:29
- **现象**：错误消息格式化后包含参数值（如 "发票号码【INV-DUP】已存在"），但测试期望的是模板消息（如 "发票号码【{}】已存在"）
- **原因**：`assertServiceException` 会将 errorCode 的消息模板用传入的 params 格式化，需传实际参数值
- **修复**：为 `assertServiceException` 添加对应的 messageParams 参数
- **影响文件**：[CrmInvoiceServiceTest.java](../../product/Server/mitedtsm-module-crm/src/test/java/com/meession/etm/module/crm/service/invoice/CrmInvoiceServiceTest.java)

### 缺陷 6：BigDecimal 精度比较失败
- **发现时间**：2026-07-14 11:29
- **现象**：`assertEquals(new BigDecimal("2000.00"), saved.getAmount())` 失败，实际值为 2000.000000
- **原因**：`assertEquals` 使用 `equals` 比较，精度不同的 BigDecimal 不相等
- **修复**：改用 `assertEquals(0, expected.compareTo(actual))` 进行数学等值比较
- **影响文件**：[CrmInvoiceServiceTest.java](../../product/Server/mitedtsm-module-crm/src/test/java/com/meession/etm/module/crm/service/invoice/CrmInvoiceServiceTest.java)

### 缺陷 7：SQL 导入时菜单 ID 冲突
- **发现时间**：2026-07-14 11:30
- **现象**：`Duplicate entry '2600' for key 'system_menu.PRIMARY'`
- **原因**：之前运行时已导入过菜单，ID 2600-2605 被占用
- **修复**：将菜单 ID 改为 6004-6009、字典类型改为 2012-2013、字典数据改为 3054-3058
- **影响文件**：[crm_invoice_init.sql](../../product/database/new/crm_invoice_init.sql)
